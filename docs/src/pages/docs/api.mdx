# JavaScript API

data-chart exposes a global `window.dataChart` object for programmatic control.

## Methods

### `dataChart.init()`

Manually initialize data-chart. Called automatically on `DOMContentLoaded`.

```js
window.dataChart.init();
```

### `dataChart.render(table)`

Render a single table as a chart.

```js
const table = document.getElementById('my-table');
window.dataChart.render(table);
```

### `dataChart.refresh(table)`

Destroy and re-render a chart. Useful after updating table data or to replay animations.

```js
const table = document.querySelector('table[data-chart]');
window.dataChart.refresh(table);
```

### `dataChart.destroy(table?)`

Remove a chart and restore the original table. If no argument, destroys all charts.

```js
// Destroy one
window.dataChart.destroy(table);

// Destroy all
window.dataChart.destroy();
```

### `dataChart.version`

```js
console.log(window.dataChart.version); // "0.1.0"
```

## Events

data-chart dispatches custom events on the `<table>` element.

### `datachart:rendered`

Fired after a chart is rendered.

```js
table.addEventListener('datachart:rendered', (e) => {
  console.log(e.detail.svg);  // SVGSVGElement
  console.log(e.detail.data); // ParsedData
});
```

### `datachart:destroyed`

Fired after a chart is destroyed.

```js
table.addEventListener('datachart:destroyed', () => {
  console.log('Chart removed');
});
```

### `datachart:error`

Fired when parsing or rendering fails.

```js
table.addEventListener('datachart:error', (e) => {
  console.error(e.detail.message);
});
```

## Error Codes

| Code | Description |
|------|-------------|
| `NO_THEAD` | No `<thead>` and `<tbody>` has no rows |
| `NO_DATA` | Zero data rows in `<tbody>` |
| `INVALID_TYPE` | Unknown `data-chart` value |

## ESM Import

```js
import dataChart from 'data-chart';

dataChart.render(document.getElementById('my-table'));
```

## TypeScript

```ts
import type { ParsedData, ChartType, ChartConfig, DataChartAPI } from 'data-chart';
```
